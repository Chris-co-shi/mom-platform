# 集成架构

## 1. 集成原则

- 外部系统只能通过 Gateway 和 Integration Hub 访问 MOM。
- 领域服务之间的即时查询和校验使用同步调用。
- 状态传播、长流程和跨系统协作优先使用领域事件。
- 所有消息消费者按“至少一次投递”设计，必须实现幂等。
- 不使用长时间 Seata 全局事务包裹完整制造流程。

## 2. 技术选择矩阵

| 场景 | 方式 | 说明 |
|---|---|---|
| 单次查询、即时校验 | OpenFeign | 设置超时、错误映射和 Trace 传播 |
| 状态变化通知 | RocketMQ 领域事件 | Outbox 可靠发布，Inbox 去重 |
| 外部同步接口 | Gateway + Integration Hub | 认证、映射、审计、限流和幂等 |
| 设备命令和结果 | RocketMQ/API | 使用 `command_id` 和状态机 |
| 模拟遥测 | MQTT | 非关键控制数据，不替代领域事件 |
| 单服务数据库事务 | 本地事务 | 默认选择 |
| 短跨库事务 PoC | Seata AT | 严格限制范围和持续时间 |
| 关键资源冻结 | TCC | 只在预占等明确资源场景评估 |
| 长业务流程 | Saga 思想 + 事件 + 补偿 | 每一步有明确状态、重试和人工处理 |

## 3. 统一事件信封

所有跨服务事件至少包含：

```json
{
  "eventId": "全局唯一事件标识",
  "eventType": "InventoryReceived",
  "eventVersion": 1,
  "occurredAt": "UTC 时间",
  "producer": "mom-wms-server",
  "aggregateType": "InventoryBatch",
  "aggregateId": "业务聚合标识",
  "correlationId": "业务关联标识",
  "workflowId": "长流程实例标识",
  "traceContext": {},
  "payload": {}
}
```

约束：

- 事件名称使用已经发生的业务事实。
- Payload 不得直接复用数据库 Entity。
- 事件版本升级必须保持消费者兼容或提供新 Topic/Type。
- 敏感信息不得进入事件。

## 4. Outbox 发布流程

```text
业务写入 + Outbox 写入（同一本地事务）
→ 提交成功
→ 发布器扫描或接收通知
→ 发送 RocketMQ
→ 标记已发布
```

必须处理：

- 应用在业务提交后、消息发送前崩溃。
- 消息发送成功但 Outbox 状态更新失败。
- 重复发送。
- 长时间未发布告警。

## 5. Inbox 消费流程

```text
接收消息
→ 检查 event_id + consumer
→ 已处理则直接确认
→ 未处理则在本地事务中写业务事实和 Inbox
→ 提交后确认消息
```

失败时允许 RocketMQ 重投，但业务处理必须保持幂等。

## 6. Integration Hub

Integration Hub 负责：

- 外部端点和 Client 配置。
- 外部模型到 MOM 契约的映射。
- 请求与响应审计。
- 幂等键和重复请求处理。
- 重试、退避、死信、人工补偿和对账。
- 入站与出站限流。
- 外部错误码与内部错误模型转换。

Integration Hub 不负责：

- 修改其他领域数据库。
- 承担 MES/WMS/QMS 核心业务规则。
- 使用一个万能流程表编排所有业务。

## 7. Feign 调用规则

- 必须设置连接和读取超时。
- 只传递白名单上下文，例如用户、Client、工厂、Trace 和关联标识。
- 禁止透传原始外部 Header。
- 失败重试只用于幂等查询或明确可重试操作。
- 写请求默认不进行客户端无条件重试。
- 调用方必须区分业务拒绝、超时、服务不可用和未知错误。

## 8. Seata 边界

允许：

- 时间短、参与资源明确、失败可快速回滚的 PoC 场景。
- 少量关键资源预留的 TCC 评估。

禁止：

- 供应商送货到成品入库的全链路事务。
- 等待人工检验、设备执行或外部系统回调的事务。
- 用 Seata 替代 Outbox、幂等、对账和人工补偿。

## 9. 错误恢复

所有跨系统流程必须定义：

- 超时。
- 重试次数和退避策略。
- 可重试与不可重试错误。
- 死信和人工处理入口。
- 对账方式。
- 最终业务状态。
- Trace、日志和告警。
