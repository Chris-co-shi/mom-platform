# 产品范围

## 1. 项目定位

MOM Platform 是面向新能源材料制造场景的工业制造运营管理平台后端。V1 以锂电池电解液为主演示产品，目标是形成可部署、可演示、可二次开发的开源工业软件底座。

项目重点不是提供通用后台 CRUD，而是验证制造业务闭环、跨系统集成、批次追溯、库存一致性、设备协同和故障恢复能力。

## 2. V1 业务闭环

```text
供应商送货
→ 原料检验
→ 原料入库
→ 生产计划与工单
→ 原料预占与领用
→ PCS 设备协同
→ 半成品与成品产出
→ 成品检验与放行
→ WCS 自动入库
→ 客户发运
→ 客诉、影响分析与模拟召回
```

## 3. 系统范围

### 3.1 本仓库负责

- Gateway：统一入口、认证、限流、审计和上下文传递。
- IAM：OAuth2.1/OIDC、用户、角色、权限、Client 和数据范围。
- MDM：集团、工厂、物料、供应商、客户和工厂模型。
- MES：生产需求、工单、版本快照、投料、生产过程和报工。
- WMS：仓库、库位、容器、收发存、预占、流水、余额和对账。
- QMS：检验、放行、不合格处置、偏差、CAPA 和变更。
- EMS：能源采集与基础统计。
- EAM：设备台账、点检、保养、维修、备件和履历。
- Integration Hub：外部接口、映射、幂等、重试、补偿与对账。
- Traceability：正向追溯、反向追溯、影响分析和模拟召回。

### 3.2 独立仓库负责

- `pcs-platform`：生产设备协同、协议适配、状态机和模拟器。
- `wcs-platform`：自动仓储调度、运输任务、状态机和模拟器。
- `mom-web`：MOM Web、供应商门户、客户门户及 Web 原型。
- `mom-mobile`：PDA、扫码、离线队列及移动端原型。
- `erp-simulator`：ERP/SAP 外部系统模拟。
- `mom-infra`：k3s、中间件、可观测性和部署脚本。

## 4. 组织和主数据边界

- 单企业、多工厂。
- V1 只完整执行一个工厂，其余工厂使用模拟数据展示集团视图。
- 物料编码集团唯一。
- 供应商和客户由集团维护。
- 工艺、配方、BOM 和质量标准由工厂维护。
- 工单下达时生成不可变生产版本快照。

## 5. 核心业务规则

- 库存采用事实流水、实时余额、预占和锁定组合模型。
- 未经质量放行的批次不得进入可用库存。
- 原料、半成品、成品通过工单、批次和容器建立多对多谱系。
- 谱系必须支持拆分、合并、返工、降级、让步接收和召回。
- 外部系统只能通过 Gateway 和 Integration Hub 访问领域能力。
- PCS 与 WCS 负责上位协同，PLC/DCS 负责硬实时控制和安全联锁。

## 6. 目标用户

- 集团和工厂管理员。
- 生产计划员、生产操作员和生产主管。
- 仓库管理员和 PDA 作业人员。
- 质量检验员和质量经理。
- 设备工程师和能源管理员。
- 供应商和客户门户用户。
- 系统集成和运维人员。

## 7. V1 成功标准

V1 必须能够在三节点 k3s 环境完成一次可重复演示，并证明：

- 完整业务闭环能够运行。
- 重复请求、消息和设备回执不会产生重复业务事实。
- 原料与成品可双向追溯。
- 服务调用和消息链路可观测。
- Redis 限流可在多 Gateway 实例间共享。
- 服务可滚动升级、回滚并完成故障恢复演练。
