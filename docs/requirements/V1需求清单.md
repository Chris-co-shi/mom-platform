# V1 需求清单

## 1. 编号规则

- `REQ-IAM`：身份与权限。
- `REQ-MDM`：主数据。
- `REQ-MES`：生产执行。
- `REQ-WMS`：仓储与库存。
- `REQ-QMS`：质量管理。
- `REQ-INT`：系统集成。
- `REQ-TRACE`：批次追溯。
- `REQ-OPS`：部署与运维。

## 2. 身份与权限

- **REQ-IAM-001**：系统应支持基于 Spring Authorization Server 的 OAuth2.1/OIDC 登录。
- **REQ-IAM-002**：Web、供应商门户和客户门户应使用 Authorization Code + PKCE。
- **REQ-IAM-003**：内部服务、ERP Simulator、PCS 和 WCS 应支持 Client Credentials。
- **REQ-IAM-004**：权限模型应支持菜单、操作权限和工厂/车间/仓库等数据范围。
- **REQ-IAM-005**：认证、授权失败和敏感操作应记录审计信息。

## 3. 主数据

- **REQ-MDM-001**：系统应支持集团、工厂、车间、产线、仓库和库区模型。
- **REQ-MDM-002**：物料编码在集团范围内唯一。
- **REQ-MDM-003**：系统应维护供应商、客户、物料、容器和计量单位基础信息。
- **REQ-MDM-004**：配方、BOM、工艺和质量标准应独立维护并支持版本。
- **REQ-MDM-005**：生产工单下达时应引用并固化不可变版本快照。

## 4. 供应商到货、质量与入库

- **REQ-WMS-001**：系统应支持创建供应商送货和原料收货任务。
- **REQ-WMS-002**：PDA 应支持扫描送货单、物料批次和容器完成收货。
- **REQ-QMS-001**：收货后应创建来料检验任务。
- **REQ-QMS-002**：未经质量放行的原料批次不得进入可用库存。
- **REQ-QMS-003**：系统应支持合格、拒收、返检、降级和让步接收结果。
- **REQ-WMS-003**：入库必须同时产生库存事实流水和实时余额变化。
- **REQ-WMS-004**：同一收货请求重复提交不得重复增加库存。

## 5. 生产执行

- **REQ-MES-001**：系统应支持创建、下达、开始、暂停、恢复和完成生产工单。
- **REQ-MES-002**：工单下达时应生成生产版本快照。
- **REQ-MES-003**：生产前应向 WMS 申请原料预占。
- **REQ-MES-004**：投料必须记录原料批次、容器、数量、工序和时间。
- **REQ-MES-005**：系统应支持半成品和成品批次产出。
- **REQ-MES-006**：PCS 重复回执不得造成重复投料、重复产出或重复报工。

## 6. 仓储与库存

- **REQ-WMS-005**：库存应区分现有量、可用量、预占量、锁定量和隔离量。
- **REQ-WMS-006**：库存余额应能够由事实流水重新计算并生成差异报告。
- **REQ-WMS-007**：不合格、冻结或过期批次不得被生产工单预占。
- **REQ-WMS-008**：成品质量放行后才可进入可用库存。
- **REQ-WMS-009**：WCS 自动入库完成后应更新容器与库位关系。

## 7. 成品质量与发运

- **REQ-QMS-004**：成品产出后应创建成品检验任务。
- **REQ-QMS-005**：系统应支持自动放行和人工放行。
- **REQ-WMS-010**：系统应支持按客户订单进行库存预占、拣货和发运。
- **REQ-WMS-011**：发运应记录客户、发运单、成品批次、数量和容器。

## 8. 集成与一致性

- **REQ-INT-001**：外部系统必须通过 Gateway 和 Integration Hub 接入。
- **REQ-INT-002**：Integration Hub 应支持请求映射、幂等、重试、死信、补偿和对账。
- **REQ-INT-003**：跨服务长流程默认使用 Outbox、RocketMQ 和 Inbox。
- **REQ-INT-004**：所有领域事件必须包含唯一 `event_id` 和业务关联标识。
- **REQ-INT-005**：系统应支持模拟超时、重复消息、乱序回执和服务重启。

## 9. 批次追溯

- **REQ-TRACE-001**：系统应支持由成品批次反查全部原料和半成品来源。
- **REQ-TRACE-002**：系统应支持由原料批次查询受影响成品、库存、发运单和客户。
- **REQ-TRACE-003**：谱系应支持拆分、合并、返工、降级和让步接收。
- **REQ-TRACE-004**：谱系边应保持不可变，纠错通过新增更正事实完成。
- **REQ-TRACE-005**：系统应支持模拟召回并生成影响范围。

## 10. 运维与展示

- **REQ-OPS-001**：系统应部署到三节点 k3s。
- **REQ-OPS-002**：HTTP、Feign、RocketMQ、定时任务和 PCS/WCS 命令应可追踪。
- **REQ-OPS-003**：Gateway 应支持 Redis 分布式限流和 429 响应。
- **REQ-OPS-004**：系统应提供 Prometheus 指标、Loki 日志、Tempo Trace 和 Grafana 仪表盘。
- **REQ-OPS-005**：系统应完成滚动升级、回滚和故障演练。
