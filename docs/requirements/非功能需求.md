# 非功能需求

## 1. 可靠性与一致性

- **NFR-REL-001**：所有写接口必须支持业务幂等键，重复提交不得产生重复业务事实。
- **NFR-REL-002**：所有 RocketMQ 消费者必须实现 Inbox 或等价去重机制。
- **NFR-REL-003**：跨服务长流程不得依赖单个长时间 Seata 全局事务。
- **NFR-REL-004**：库存余额、预占和流水必须支持离线重算与差异对账。
- **NFR-REL-005**：PCS/WCS 命令必须具有 `command_id`、状态机和超时恢复机制。

## 2. 性能目标

以下目标用于 V1 测试，不代表最终生产容量：

- 普通查询接口 P95 小于 500 ms。
- 普通写接口 P95 小于 800 ms，不含外部设备等待时间。
- Gateway 在三实例下应完成共享限流验证。
- 单批次正向或反向追溯在 10 万条谱系边的测试数据下 P95 小于 2 秒。
- 库存流水写入应支持至少 100 TPS 的稳定压测。
- 批量导出和复杂追溯应使用成本型限流，不与普通接口共享同等令牌成本。

## 3. 安全

- **NFR-SEC-001**：禁止使用 OAuth2 密码授权模式。
- **NFR-SEC-002**：Web 和 PDA 使用 Authorization Code + PKCE；机器调用使用 Client Credentials。
- **NFR-SEC-003**：服务端必须校验 Token、Audience、Scope 和权限。
- **NFR-SEC-004**：权限应同时支持功能权限和工厂、车间、仓库、产线等数据范围。
- **NFR-SEC-005**：密码、Token、密钥、SDS 敏感字段等不得进入日志或 Trace 属性。
- **NFR-SEC-006**：外部接口应支持 Client、IP 白名单、签名或双向 TLS 的扩展点。

## 4. 可观测性

- **NFR-OBS-001**：HTTP、Gateway、Feign、RocketMQ、定时任务和设备命令链路必须传播 Trace 上下文。
- **NFR-OBS-002**：日志必须包含 `trace_id`、`span_id`，长业务流程还应包含 `correlation_id` 或 `workflow_id`。
- **NFR-OBS-003**：Prometheus 标签禁止使用用户 ID、Trace ID 等高基数字段。
- **NFR-OBS-004**：错误日志应能够跳转到对应 Trace，Trace 应能够关联日志。
- **NFR-OBS-005**：关键业务动作应输出低基数业务指标，例如收货成功数、消息重试数和限流拒绝数。

## 5. 可维护性

- **NFR-MNT-001**：构建必须使用 JDK 25 和 Maven 3.9.9 以上版本。
- **NFR-MNT-002**：业务代码禁止新增 `javax.*` 依赖。
- **NFR-MNT-003**：`*-api` 不得暴露数据库 Entity、Mapper 或 Repository。
- **NFR-MNT-004**：领域服务禁止依赖其他领域的 `*-server` 模块。
- **NFR-MNT-005**：关键边界使用 ArchUnit 或等价自动化测试约束。
- **NFR-MNT-006**：所有数据库变更必须由 Flyway 管理，禁止人工漂移。

## 6. 可部署性

- **NFR-DEP-001**：应用应支持容器化并部署到三节点 k3s。
- **NFR-DEP-002**：应用应提供启动、存活和就绪探针。
- **NFR-DEP-003**：配置与密钥必须分离，密钥不得提交到 Git。
- **NFR-DEP-004**：核心服务应支持滚动升级和版本回滚。
- **NFR-DEP-005**：数据库迁移必须支持向前兼容的滚动发布策略。

## 7. 数据与审计

- **NFR-DATA-001**：PostgreSQL 每服务使用独立 Schema，禁止跨 Schema JOIN 和跨域写入。
- **NFR-DATA-002**：时间戳统一以 UTC 存储，业务日期和工厂时区单独表达。
- **NFR-DATA-003**：库存流水、谱系边、审计记录等关键事实原则上不可物理覆盖。
- **NFR-DATA-004**：敏感操作必须记录操作者、时间、来源、前后状态和关联业务单号。

## 8. 故障恢复

V1 必须演练：

- Redis 暂时不可用。
- RocketMQ 重复投递和消费重试。
- 服务处理完成但确认消息失败。
- PCS/WCS 超时、断线、重复回执和乱序回执。
- Nacos 短时不可用。
- Pod 重启、节点迁移、滚动升级和回滚。
- WCS 设备堵塞与人工接管。
