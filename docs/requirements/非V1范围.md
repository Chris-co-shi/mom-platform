# 非 V1 范围

以下能力明确不进入一个月 V1。除非通过新的 ADR 和路线图变更正式批准，否则不得在 V1 开发中扩展这些内容。

## 1. 业务能力

- AI Agent 业务能力。
- 完整 APS 高级计划与排程优化算法。
- 供应商合同、询报价、采购发票和财务对账。
- 客户合同、销售报价、开票和应收管理。
- 完整实验室信息管理系统（LIMS）。
- 完整法规申报和危化品合规平台。
- 预测性维护和复杂设备健康算法。
- 多个工厂同时运行完整执行链。

## 2. 数据平台

- Kafka。
- Flink。
- IoTDB。
- Elasticsearch。
- 数据湖、湖仓和大规模 CDC 平台。
- 完整实时 OEE、能耗和质量流计算。

V1 只使用 MQTT Broker、RocketMQ 和 PostgreSQL。后续只有在高频遥测、事件重放、实时分析或检索压力出现后，才评估引入上述组件。

## 3. 工作流与规则平台

- Flowable 等通用 BPMN 引擎。
- 通用低代码平台。
- 可视化复杂规则编排平台。
- 面向所有领域的统一流程中心。

V1 的关键业务流程由领域状态机、领域事件和应用服务明确表达。

## 4. 工业设备

- 真实 PLC、DCS 和工业网关接入。
- 真实产线安全联锁。
- 真实 AGV、堆垛机、输送线和提升机控制。
- 硬实时控制逻辑。

V1 全部使用软件模拟器，并支持断线、超时、重复回执、报警、堵塞和人工接管。

## 5. 平台规模

- 跨地域多活。
- 超大规模租户隔离。
- 公有云 SaaS 商业化能力。
- 百万设备接入。
- PB 级历史数据归档。

## 6. 前端范围

V1 仅要求：

- MOM Web 最小闭环。
- PDA 最小闭环。
- 供应商门户最小闭环。
- 客户门户最小闭环。

不要求完整视觉设计系统、复杂大屏平台或所有业务模块的完整页面。

## 7. 变更规则

需要把本文件中的能力提前到 V1 时，必须同时完成：

1. 说明业务价值和必要性。
2. 评估对一个月交付目标的影响。
3. 创建或更新 ADR。
4. 修改 V1 路线图和验收标准。
5. 明确要从当前 V1 范围中移除的等量工作。
