# 领域术语表

| 术语 | 英文/缩写 | 定义 | 权威领域 |
|---|---|---|---|
| 制造运营管理 | MOM | 连接计划、生产、质量、仓储、设备和集成的制造运营平台 | 平台整体 |
| 制造执行系统 | MES | 管理工单、工艺版本、投料、生产过程、报工和批次产出 | MES |
| 仓库管理系统 | WMS | 管理仓库、库位、容器、库存、收发存和预占 | WMS |
| 质量管理系统 | QMS | 管理检验、放行、不合格处置、偏差和 CAPA | QMS |
| 生产控制系统 | PCS | 负责生产设备上位协同、命令状态机和协议适配 | PCS |
| 仓库控制系统 | WCS | 负责自动仓储运输任务和设备协同 | WCS |
| 主数据管理 | MDM | 管理集团、工厂、物料、供应商、客户和版本索引 | MDM |
| 企业资产管理 | EAM | 管理设备台账、点检、保养、维修和备件 | EAM |
| 能源管理系统 | EMS | 管理水、电、气等能源采集和统计 | EMS |
| 集成中心 | Integration Hub | 统一外部接口、映射、幂等、重试、补偿和对账 | Integration |
| 工单 | Production Order | 一次生产任务及其范围、数量和状态 | MES |
| 生产版本快照 | Production Version Snapshot | 工单下达时固化的配方、BOM、工艺和质量标准版本 | MES |
| 批次 | Batch/Lot | 具有共同生产或接收属性、可被追溯的一组物料 | MES/WMS |
| 容器 | Handling Unit/Container | 承载批次及数量的物理或逻辑载体 | WMS |
| 批次谱系 | Batch Genealogy | 原料、半成品、成品在生产过程中的多对多关系 | Traceability |
| 谱系边 | Genealogy Edge | 描述一个输入批次与输出批次之间生产关系的不可变事实 | Traceability |
| 库存事实流水 | Inventory Ledger | 每次收、发、移、锁、解锁、调整产生的不可变库存事实 | WMS |
| 库存余额 | Inventory Balance | 根据库存事实维护的当前数量视图 | WMS |
| 预占 | Reservation | 为特定工单或发运需求保留库存数量 | WMS |
| 锁定 | Lock | 因质量、异常或业务控制禁止使用的库存数量 | WMS/QMS |
| 隔离库存 | Quarantine Stock | 尚未放行或存在质量风险的库存 | WMS/QMS |
| 放行 | Release | 质量部门确认批次可进入下一业务状态 | QMS |
| 让步接收 | Concession Acceptance | 在受控条件下接受存在偏差但仍可使用的物料 | QMS |
| 领域事件 | Domain Event | 已经发生且需要通知其他领域的业务事实 | 各领域 |
| Outbox | Transactional Outbox | 业务事务内记录待发布消息，事务后可靠发送 | Integration/Framework |
| Inbox | Consumer Inbox | 消费端记录消息处理结果以避免重复处理 | Integration/Framework |
| 关联标识 | Correlation ID | 跨多个技术 Trace 关联同一业务流程的标识 | Framework |
| 工作流标识 | Workflow ID | 标识一个长时间业务流程实例 | Framework |
| 事件标识 | Event ID | 领域事件的全局唯一标识，用于幂等和追踪 | Framework |
| 命令标识 | Command ID | PCS/WCS 设备命令的唯一标识 | PCS/WCS |

## 术语使用约束

- API、事件、数据库和文档应使用同一术语，禁止同义词随意混用。
- “批次”表示业务追溯单元；“容器”表示承载批次的载体，两者不得合并为同一模型。
- `trace_id` 是技术调用链标识，不能替代 `correlation_id`、`workflow_id` 或业务单号。
- 术语定义发生变化时，应先更新本表和相关 ADR，再修改代码。
