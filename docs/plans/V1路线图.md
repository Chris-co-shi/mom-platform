# V1 路线图

## 1. 总目标

在一个月内交付一个可部署、可重复演示的工业 MOM V1，完成供应商送货到客户发运及批次追溯的最小完整闭环，并在三节点 k3s 环境完成可观测性、滚动升级、回滚和故障恢复演练。

## 2. 阶段划分

| 阶段 | 目标 | 主要输出 |
|---|---|---|
| Phase 01 | 技术骨架可运行 | JDK 25、Boot 4、Gateway、IAM、MDM、Integration、数据库、中间件、Trace、限流 |
| Phase 02 | 到货、检验和入库闭环 | 供应商送货、来料检验、PDA 收货、库存流水和余额 |
| Phase 03 | 生产执行和 PCS 协同闭环 | 工单、版本快照、预占、投料、半成品/成品、PCS 模拟器 |
| Phase 04 | 放行、WCS 入库、发运和追溯闭环 | 成品放行、自动入库、客户发运、正反向追溯、模拟召回 |

## 3. Phase 01：技术骨架

### 目标

证明 JDK 25、Spring Boot 4.1、Spring Cloud 2025.1 和 Spring Cloud Alibaba 2025.1 组合可以支撑项目基础设施。

### 输出

- Maven Reactor 和统一依赖管理。
- Gateway、IAM、MDM、Integration 最小启动应用。
- PostgreSQL 每服务独立 Schema 和 Flyway。
- Nacos 注册发现和配置导入。
- RocketMQ 生产、消费和 Trace 上下文传播。
- Redis 限流和幂等基础能力。
- OpenTelemetry Collector、Tempo、Prometheus、Loki 和 Grafana。
- Testcontainers 集成测试和 ArchUnit 依赖约束。

### 完成定义

- 所有模块在 JDK 25 下执行 `mvn verify` 成功。
- 一个请求能够经过 Gateway 调用后端服务。
- Grafana 能查看对应 Trace 和日志。
- 三个 Gateway 实例共享 Redis 限流状态。
- RocketMQ 消息能够传播关联标识和 Trace 上下文。

## 4. Phase 02：供应商到货、质量与入库

### 输出

- 供应商、物料、工厂、仓库、库位和容器最小主数据。
- 送货单、收货任务和来料检验任务。
- PDA 扫码收货与离线队列最小实现。
- 检验合格、拒收、降级和让步接收。
- 库存事实流水、实时余额、质量状态和幂等提交。

### 完成定义

同一收货请求重复提交三次，只产生一次收货事实、一次库存流水和一次余额增加；未经质量放行的批次不可进入可用库存。

## 5. Phase 03：生产执行与 PCS

### 输出

- 配方、BOM、工艺和质量标准版本索引。
- 生产工单和不可变生产版本快照。
- 原料预占、领用、投料和消耗。
- PCS 命令协议、状态机和软件模拟器。
- 半成品、成品批次和容器产出。
- 投料和产出谱系边。

### 完成定义

PCS 断线、超时和重复回执后，工单能够恢复并继续执行，且库存、投料和产出不重复。

## 6. Phase 04：成品放行、自动仓储和追溯

### 输出

- 成品检验与自动/人工放行。
- WCS 自动入库请求、运输任务和模拟设备。
- 成品库存、客户订单、拣货和发运。
- 正向追溯、反向追溯、影响分析和模拟召回。
- k3s 部署、滚动升级、回滚和故障演练报告。

### 完成定义

可以由任一成品批次查到所有原料来源，也可以由任一原料批次查到受影响成品、当前库存、发运单和客户。

## 7. 里程碑门禁

每个 Phase 进入下一阶段前必须通过：

1. 需求验收。
2. 自动化测试。
3. 架构边界检查。
4. 数据迁移检查。
5. Trace、日志和指标检查。
6. 文档更新。

## 8. 延期处理原则

若阶段延期，优先减少非核心页面和模拟数据丰富度，不得删除以下能力：

- 库存一致性。
- 批次谱系。
- Outbox/Inbox 与幂等。
- PCS/WCS 状态机。
- 链路追踪。
- Redis 限流。
- 故障恢复验收。
