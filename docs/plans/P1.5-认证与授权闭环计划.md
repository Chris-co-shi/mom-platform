## 5. P1.5 Definition of Done

1. 四个 Public Client、三种 user_type 与应用访问矩阵实现并通过 E2E。
2. Web/Mobile 使用 Authorization Code + PKCE，密码只提交 IAM 页面。
3. Access Token、Opaque Refresh Token、ID Token 生命周期符合基线。
4. Rotation、重放检测、Session 撤销、revoked sid 与 Redis Fail Closed 已验证。
5. Gateway 和业务服务职责边界实现并测试。
6. RBAC、Permission、Factory/Party Scope、对象归属与 404 防枚举通过测试。
7. `/api/iam/me` 成为 Web/Mobile 权限上下文来源。
8. CurrentActor、强制审计字段、显式 SYSTEM Actor 和乐观锁已实现。
9. IAM 管理 API、管理页面、安全审计和 Session 管理可用。
10. Web Token 不持久化；Mobile Refresh Token 使用 Android 安全存储。
11. 所有正式 PostgreSQL Flyway DDL 有完整中文表、字段、枚举、约束、索引、JSON、UTC 和安全边界注释。
12. 三仓库文档、代码、Claims、Client ID、user_type 与阶段状态一致。
