# V1 垂直切片计划

## 1. 拆分原则

- 以可演示业务结果拆分，不以 Controller、Service、Mapper 分层拆分。
- 每个切片必须同时包含需求、数据、接口、事件、异常路径、可观测性和验收。
- 每个 Web/PDA 切片必须先完成用户流程、页面状态、原型和 API 映射。
- 每个跨服务切片必须明确权威数据归属和一致性策略。

## 3. VS-00：技术骨架闭环

输入：合法用户或 Client Token。

流程：

```text
Client → Gateway → 后端服务 → PostgreSQL/Redis/RocketMQ
                  ↓
           OpenTelemetry Collector → Tempo/Loki/Prometheus
```

验收：认证、路由、限流、Trace、日志、指标和消息上下文均可用。

## 4. VS-01A：供应商送货与来料检验

### 主流程

1. ERP Simulator 或供应商门户创建供应商送货通知；通知本身不产生实物库存。
2. Integration Hub 完成映射和幂等校验，WMS 创建收货任务。
3. PDA 或受控接口完成实物收货，WMS 创建收货批次、容器、库存流水与余额，初始质量状态固定为 `INSPECTION_HOLD`。
4. WMS 发布 `MaterialReceived` 已发生事实事件，QMS 据此创建来料检验任务。
5. QMS 完成检验和质量处置，发布 `InspectionDispositionDecided` 已发生事实事件。
6. WMS 消费处置事件，将库存质量状态调整为 `AVAILABLE`、`REJECTED`、`CONCESSION` 或 `DOWNGRADED`。

### 关键事件

- `SupplierDeliveryNotified`
- `ReceivingTaskCreated`
- `MaterialReceived`
- `IncomingInspectionCreated`
- `InspectionDispositionDecided`

### 验收

相同外部请求重复三次，只创建一个送货通知、一个收货任务、一次收货事实和一个检验任务；QMS 不直接修改 WMS 数据库，未检验或未放行库存可以存在但不可作为可用库存。

## 5. VS-01B：PDA 收货与原料入库

### 主流程

1. PDA 扫描送货单、物料批次和容器。
2. 校验物料、供应商、数量、质量状态和库区限制。
3. 创建库存流水并更新余额，初始质量状态只能是 `INSPECTION_HOLD`；不得在人工检验前制造已通过状态。
4. 打印或模拟打印批次标签。

### 异常流程

- 网络断开：进入离线队列。
- 重复扫码：前端提示并由后端幂等拦截。
- 检验未放行：库存事实保留，但禁止进入 `AVAILABLE` 可用库存。
- 数量冲突：进入人工处理。

## 6. VS-01C：工单下达与原料预占

### 主流程

1. MES 创建并下达生产工单。
2. 固化配方、BOM、工艺和质量标准快照。
3. MES 请求 WMS 预占原料。
4. WMS 按批次状态、有效期、库区和先进先出规则分配库存。

### 验收

- 工单快照创建后不可被主数据版本修改影响。
- 不合格、锁定、隔离和过期批次不可预占。
- 重复预占请求不会重复占用库存。

## 7. VS-01D：PCS 投料、混合与灌装

### 主流程

1. MES 向 PCS 发送投料命令。
2. PCS 模拟称重、投料、反应、转移和灌装。
3. MES 记录原料消耗、过程参数和产出。
4. Traceability 创建输入批次到输出批次的谱系边。

### 异常流程

- PCS 断线、超时、重复回执、乱序回执。
- 设备报警和人工接管。
- 实际投料量与计划量偏差。

### 验收

所有异常恢复后只产生一次有效投料、一次有效产出和一组有效谱系边。

## 8. VS-01E：成品检验与放行

- 成品产出后自动创建检验任务。
- 未放行成品进入隔离库存。
- 自动或人工放行后转为可用状态。
- 不合格成品支持返工、报废、降级和重新检验。

## 9. VS-01F：WCS 自动入库

- WMS 创建入库请求。
- WCS 生成运输任务和路由。
- 模拟输送线、提升机、堆垛机和库位。
- 完成后更新容器和库位关系。
- 堵塞、超时和人工接管必须可恢复。

## 10. VS-01G：客户发运与批次追溯

- 客户订单触发库存预占、拣货和发运。
- 发运记录绑定客户、发运单、容器和成品批次。
- 支持成品到原料的反向追溯。
- 支持原料到成品、库存、发运单和客户的正向追溯。

## 11. VS-01H：模拟召回与故障演练

- 指定一个原料批次发起模拟召回。
- 输出在库、在制、已发运和客户影响范围。
- 演练 Redis 故障、MQ 重复投递、Pod 重启、PCS/WCS 断线和版本回滚。
- 形成故障现象、告警、Trace、恢复动作和最终数据校验报告。

## 12. 每个切片完成定义

- 需求编号已关联。
- 页面原型和状态矩阵已完成。
- API、命令和事件契约已评审。
- 数据迁移已提交。
- 正常和异常路径测试通过。
- Trace、日志和业务指标可查询。
- 文档、ADR 和代码保持一致。
