## S02 已交付并合并

- 独立 PostgreSQL Schema：`mom_iam`。
- 7 个连续 Flyway 迁移，创建 16 张正式表并初始化环境无关目录。
- 用户、内部资料、Party Binding、RBAC、Factory Scope、Mobile Access 和 Client Policy。
- Spring Security 7.1 Authorization Server 官方 JDBC 协议表。
- 用户授权 Session、Refresh Token 轮换状态和追加型安全审计。
- 13 个 MOM 持久化 Entity、10+ 个稳定 Enum、13 个 Mapper 和受控 Repository。
- 3 个内置角色、28 个 IAM Permission、59 条 Role-Permission 关系和 4 个 Client Policy。
- PostgreSQL Catalog 中文注释检查、约束/索引测试、官方 JDBC Schema 兼容测试。
- Slice 01 CurrentActor、强制审计、SYSTEM Actor、乐观锁和 Wrapper-only Update 门禁复用。

架构说明：[IAM 数据库与领域模型](../architecture/IAM数据库与领域模型.md)。

## 当前 S03 边界

S03 实现登录、密码认证、Authorization Server 标准端点、Authorization Code、PKCE S256、OIDC、JWT/JWK 签发基础、四个 Public Client、Client 与 `user_type` 入口隔离及首次改密。

S03 不提前实现 S04 的完整 RBAC/Factory/Party Claims、`/api/iam/me`，也不提前实现 S05 的自定义 Refresh Rotation、重放检测、Session 撤销或 Redis revoked sid。
