## S05 已在阶段分支交付

- Web Session 绝对有效期 8 小时，Mobile Session 绝对有效期 12 小时；Refresh 不延长绝对有效期。
- 初始授权码换 Token 创建 `iam_user_session` 和 sequence=1 的 Opaque Refresh 状态。
- Refresh Token 使用 48 字节安全随机数，仅在 HTTPS Token 响应返回一次；数据库只保存 HMAC-SHA-256 十六进制摘要。
- Spring Authorization Server JDBC Authorization 表不保存 Refresh Token 明文。
- 四个 Public Client 使用标准 `grant_type=refresh_token`；专用 Public Refresh Client 认证不伪造 `code_verifier`。
- Rotation 使用 PostgreSQL `FOR UPDATE` 行锁，先标记旧 Token 为 ROTATED，再创建唯一 ACTIVE 后继并维护 sequence/replacement。
- 旧 ROTATED Token 无宽限窗口；重放立即将 Session 标记为 COMPROMISED，撤销当前 ACTIVE Token，并写入 Redis revoked sid。
- Redis revoked sid 读取或写入故障抛出专用异常，调用方不得把故障解释为未撤销。
- JWT Access Token 加入 `sid`；Web/Mobile 绝对时长、单 ACTIVE、摘要不等于明文、Rotation、Replay、COMPROMISED、Redis 标记与 Fail Closed 均有 PostgreSQL + Redis 测试。
- 完整 Maven Reactor、Nacos/Redis、PostgreSQL packaged smoke、Messaging、Seata、Observability 与 Observability Stack CI 全部通过。
