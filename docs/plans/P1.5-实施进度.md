## S06 已在阶段分支交付

- `mom-security` 定义稳定 JWT Claims、四个 Public Client、不可变授权上下文和 Role/Permission Authority 映射；OAuth Scope 与业务 Permission 保持分离。
- Gateway 和业务 Resource Server 均校验签名、Issuer、有效期、Audience、`client_id` 与 Audience 一致性，以及 `sid`、`jti`、`user_type`、Role、Permission、Factory 和 Party Claim 形态。
- Gateway 对 Admin、Mobile、Supplier、Customer Client 实施粗粒度路由隔离；未知 `/api/**` 默认拒绝。
- Gateway 删除客户端伪造的全部 `X-MOM-*` 身份 Header，保留原始 Bearer Token 和 `X-Factory-Id` 工作上下文。
- Gateway 查询 Redis revoked sid；已撤销返回 401，Redis 超时或故障返回 503，受保护请求 Fail Closed。
- 业务服务可显式启用 Servlet Resource Server，独立验证 Bearer JWT，并最终校验 Permission、Current Factory、Party Scope 与对象归属。
- 对其他 Factory、Supplier 或 Customer 不可见的对象统一使用 404 契约，避免对象枚举。
- 旧 Nacos、限流和可观测性 Smoke 仅在测试进程显式关闭 Gateway Auth；生产默认仍启用 JWT/撤销安全链。
- 完整 Maven Reactor、Gateway/Resource Server 测试、Nacos/Redis、PostgreSQL packaged smoke、Messaging、Seata、Observability 与 Observability Stack CI 全部通过。

## 当前 S07 边界

S07 实现 IAM 管理 API：用户、角色、Permission 只读目录、Factory Scope、Mobile Access、Session、安全审计、OAuth Client 查看与受控启停、外部主体重新绑定，以及至少一个有效 `PLATFORM_ADMIN` 的安全约束。

S07 不实现 Web/Mobile 客户端运行时、MOM Admin 页面或 Supplier/Customer Portal 页面。
