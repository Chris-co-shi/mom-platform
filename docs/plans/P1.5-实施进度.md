## S03～S07 已交付

- Authorization Server、OIDC、Authorization Code + PKCE S256、四个 Public Client 与数据库账号认证。
- 有效 Role/Permission、Factory/Party Scope、JWT 业务 Claims、Bearer `/api/iam/me`、Current Factory 与统一 404 基础契约。
- Web 8 小时、Mobile 12 小时绝对 Session；Opaque Refresh、HMAC 摘要、单 ACTIVE Rotation、重放检测、COMPROMISED、Redis revoked sid 与 Fail Closed。
- Gateway 与业务 Resource Server 双重 JWT 校验、Client 路由隔离、伪造身份 Header 清理、Bearer 转发、Permission/Factory/Party 最终授权和 404 防枚举。
- `/api/iam/admin/**` 提供用户、角色、Permission、Factory Scope、Mobile Access、Session、安全审计、OAuth Client 与外部 Party Binding 管理能力。
- 用户授权与角色 Permission 读取分别返回 `userVersion`、`roleVersion`；Role、Factory、Mobile、Party 与 Permission 全量替换命令必须提交读取版本，成功后父聚合版本原子加一并返回完整最新快照。
- 过期读取版本稳定返回 HTTP 409 `stale_version`，关系、父版本和成功审计在同一 PostgreSQL 事务中保持零副作用；最后一个 `PLATFORM_ADMIN` 与内置角色保护继续有效。
- 高风险写操作逐项校验 Permission，管理员自我禁用/删除、最后一个有效 `PLATFORM_ADMIN`、跨 `user_type` 角色分配和外部 Factory Scope 均 Fail Closed。
- 用户或 Client 禁用会撤销 PostgreSQL Session/Refresh 状态并写入 Redis revoked sid。

## S08 已完成

- `mom-web` 实现 `@mom/auth`、`@mom/access`、`@mom/api-client` 与三应用独立 Public Client。
- Access、Refresh、ID Token 仅内存保存；`sessionStorage` 仅保存一次性 PKCE 事务并在使用后删除。
- Callback 校验 state、nonce、RS256、Issuer、Audience、Expiry，并清理地址栏协议参数。
- 401 使用 Single Flight，每业务请求最多重试一次；403 不刷新。
- `/api/iam/me` 是完整权限上下文来源；当前 Factory 偏好重新校验；外部门户不提供 Party 切换。
- Web CI 已加入 concurrency 与 TypeScript 失败诊断；TypeScript 6、Web Crypto、真实认证测试和三应用构建全部通过。

## 当前 S09 边界

S09 的 S07 后端契约阻塞已在 `mom-platform` Exact SHA `bd06d179eadd026dbd517d00c0251c11dd6b7c9a` 解除。`mom-web` 在 MOM Admin 中实现用户、角色、Permission、Factory Scope、Mobile Access、外部主体绑定、Session、安全审计和 OAuth Client 管理页面；`mom-platform` 保持加固后的读取版本、版本化写入、409 与完整快照契约稳定并提供联调支持。

S09 不修改 IAM Schema、授权协议或服务端最终授权语义，不实现 Supplier/Customer 正式业务页面或 Mobile Auth Runtime。
