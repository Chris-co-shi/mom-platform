## S03～S06 已交付

- Authorization Server、OIDC、Authorization Code + PKCE S256、四个 Public Client 与数据库账号认证。
- 有效 Role/Permission/Factory/Party Scope、JWT 业务 Claims、Bearer `/api/iam/me`、Current Factory 与统一 404 基础契约。
- Web 8 小时、Mobile 12 小时绝对 Session；Opaque Refresh、HMAC 摘要、单 ACTIVE Rotation、重放检测、COMPROMISED、Redis revoked sid 与 Fail Closed。
- Gateway 与业务 Resource Server 双重 JWT 校验、Client 路由隔离、伪造身份 Header 清理、Bearer 转发、Permission/Factory/Party 最终授权和 404 防枚举。

## S07 已在阶段分支交付

- `/api/iam/admin/**` 提供用户、角色、Permission 只读目录、Factory Scope、Mobile Access、Session、安全审计、OAuth Client 与外部 Party Binding 管理能力。
- 高风险写操作逐项校验 Permission，管理员自我禁用/删除、最后一个有效 `PLATFORM_ADMIN`、跨 `user_type` 角色分配和外部 Factory Scope 均 Fail Closed。
- 用户或 Client 禁用会复用 S05 权威服务撤销 PostgreSQL Session/Refresh 状态并写入 Redis revoked sid。
- 安全审计保持追加写入，响应和审计摘要不包含密码摘要、Token、授权码、`code_verifier` 或私钥。
- `mom.iam.admin.enabled` 必须显式开启；只有 Servlet 应用存在 `JdbcTemplate` 时 IAM Admin AutoConfiguration 才整体生效。
- MDM、Integration、IAM 无基础设施启动、IAM Admin Disabled、IAM Admin Enabled + JdbcTemplate 和 S07 PostgreSQL + Redis 管理 API E2E 均已覆盖。
- 完整 Maven Reactor、Nacos/Redis、PostgreSQL packaged smoke、Messaging、Seata、Observability 与 Observability Stack CI 全部通过。

## 当前 S08 边界

S08 由 `mom-platform` 核验 OAuth/OIDC、Token、Refresh、Logout 和 `/api/iam/me` 契约；由 `mom-web` 实现共享 `@mom/auth`、`@mom/access`、`@mom/api-client`、三应用 PKCE、内存 Access Token、Single Flight Refresh、启动恢复和统一 401/403/404 处理。

S08 不实现 MOM Admin 权限管理页面、Supplier/Customer 业务页面或 Mobile Auth Runtime。
