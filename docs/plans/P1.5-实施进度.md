## S03 已交付

- Spring Security 7.1 Authorization Server 与 OpenID Connect Discovery、UserInfo、JWK Set 和 OIDC Logout 基础端点。
- Authorization Code + PKCE；缺失 Challenge 或 `plain` 在进入标准端点前拒绝。
- 四个 Public Client 使用 JDBC Registered Client Store 幂等同步；无 `client_secret`，只允许 Authorization Code。
- Redirect URI 与 Post Logout Redirect URI 从环境配置读取并精确匹配。
- IAM 自有登录页、首次改密页、CSRF、HttpOnly IAM Session Cookie 与 Session Fixation 防护。
- Delegating PasswordEncoder（默认 BCrypt）、禁用/锁定账号拒绝、失败累计、临时锁定和成功清零。
- Client Policy、`user_type`、外部 Party Binding 与 Mobile Access 在签发 Code 前校验。
- JDBC Authorization/Consent Store、RS256、固定 `kid`、JWK Set、10 分钟 JWT Access Token 和 OIDC ID Token。

## S04 已在阶段分支交付

- 有效 Role 只计算启用、未删除、用户类型匹配且处于有效期的分配。
- Permission 为全部有效角色的启用 Permission 并集，不实现继承、Deny 或 ABAC。
- Factory Scope 与外部 Party Binding 按状态和有效期计算并稳定排序。
- Access Token 已加入 `roles`、`permissions`、`factory_ids`，外部用户加入 `party_type`、`party_id`。
- `/api/iam/me` 使用 Bearer JWT，并重新读取 IAM 权威数据作为前端权限上下文来源。
- `X-Factory-Id` 只作为当前工作上下文，必须属于有效 Factory Scope。
- 业务对象 Factory/Party 不可见时统一抛出 404，避免对象枚举。
- JWT 列表 Claims 使用稳定可序列化类型，兼容 Spring Authorization Server JDBC Store。
- 真实 PostgreSQL 覆盖内部与供应商授权上下文、过期/禁用关系、Token Claims、Me API、403 和 404 契约。
- Maven Reactor、Nacos/Redis、PostgreSQL packaged smoke、Messaging、Seata、Observability 与 Observability Stack CI 全部通过。

## 当前 S05 边界

S05 实现 Web 8 小时、Mobile 12 小时绝对用户授权 Session，高熵 Opaque Refresh Token、HMAC-SHA-256 摘要、单 ACTIVE Token、事务行锁 Rotation、旧 Token 重放后的 COMPROMISED、Redis revoked sid、Session 撤销与 Redis Fail Closed。

S05 不修改 Gateway/业务服务 Resource Server，不实现 IAM 管理 API，不实现 Web/Mobile 客户端运行时，也不把 Refresh Token 明文或摘要写入日志、Trace 或审计。
