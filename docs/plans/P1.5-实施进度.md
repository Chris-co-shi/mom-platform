## S02 已交付并合并

- 独立 PostgreSQL Schema：`mom_iam`。
- 7 个连续 Flyway 迁移，创建 16 张正式表并初始化环境无关目录。
- 用户、内部资料、Party Binding、RBAC、Factory Scope、Mobile Access 和 Client Policy。
- Spring Security 7.1 Authorization Server 官方 JDBC 协议表。
- 用户授权 Session、Refresh Token 轮换状态和追加型安全审计。
- 13 个 MOM 持久化 Entity、10+ 个稳定 Enum、13 个 Mapper 和受控 Repository。
- 3 个内置角色、28 个 IAM Permission、59 条 Role-Permission 关系和 4 个 Client Policy。
- PostgreSQL Catalog 中文注释检查、约束/索引测试、官方 JDBC Schema 兼容测试。
- Slice 01 CurrentActor、强制审计、SYSTEM Actor、乐观锁和 Wrapper-only Update 门禁复用。

架构说明：[IAM 数据库与领域模型](../architecture/IAM数据库与领域模型.md)。

## S03 已在阶段分支交付

- Spring Security 7.1 Authorization Server 与 OpenID Connect Discovery、UserInfo、JWK Set 和 OIDC Logout 基础端点。
- Authorization Code + PKCE；缺失 Challenge 或 `plain` 在进入标准端点前拒绝。
- 四个 Public Client 使用 JDBC Registered Client Store 幂等同步；无 `client_secret`，只允许 Authorization Code。
- Redirect URI 与 Post Logout Redirect URI 从环境配置读取并精确匹配。
- IAM 自有登录页、首次改密页、CSRF、HttpOnly IAM Session Cookie 与 Session Fixation 防护。
- Delegating PasswordEncoder（默认 BCrypt）、禁用/锁定账号拒绝、失败累计、临时锁定和成功清零。
- Client Policy、`user_type`、外部 Party Binding 与 Mobile Access 在签发 Code 前校验。
- JDBC Authorization/Consent Store、RS256、固定 `kid`、JWK Set、10 分钟 JWT Access Token 和 OIDC ID Token。
- JWT 只加入 `client_id`、`user_type` 等 S03 基础信息；完整 Roles、Permissions、Factory、Party Claims 留给 S04。
- S05 自定义 Refresh Rotation、重放检测、Session 撤销与 revoked sid 未提前实现。
- 全量 Maven Reactor、Nacos/Redis、PostgreSQL、Messaging、Seata、Observability 与 Observability Stack CI 通过。

## 当前 S04 边界

S04 只实现授权上下文读取与表达：有效 Role、Permission、Factory Scope、Party Binding、JWT 业务 Claims、`/api/iam/me`、Current Factory 校验和对象不可见时统一 404 的基础服务端契约。

S04 不创建用户授权 Session，不签发自定义 Refresh Token，不实现 Rotation/replay/revoked sid，不修改 Gateway 或业务服务 Resource Server，也不实现 IAM 管理 API。
